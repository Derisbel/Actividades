1-¿Qué es un Dato?

Un dato es una representación simbólica (numérica, alfabética, etc.) de un atributo o característica de una entidad. En términos simples, son hechos básicos que describen algo específico, por ejemplo, la edad de una persona, el nombre de un producto, etc.

2-¿Qué es información?

La información es el resultado del procesamiento, organización e interpretación de datos en un contexto específico. Es decir, los datos que han sido procesados y organizados de manera que sean útiles y significativos.

3-¿Diferencia entre dato e información?

La diferencia clave radica en el contexto y el significado. Un dato es un hecho aislado que puede no tener sentido por sí solo, mientras que la información es el resultado del análisis de datos que proporciona significado y contexto.

4-¿Tipos de datos?

Los tipos de datos pueden clasificarse en:

  • Datos numéricos: Incluyen números enteros o decimales.
  • Datos alfabéticos: Son letras o palabras.
  • Datos alfanuméricos: Combinan letras y números.
  • Datos lógicos: Representan verdadero/falso o 1/0.
  • Datos de fecha/hora: Indican fechas y tiempos específicos.

    • 5-¿Qué es un modelado de Datos?

    El modelado de datos es el proceso de diseñar la estructura y relaciones de una base de datos, utilizando técnicas y herramientas específicas para representar cómo se almacenarán y organizarán los datos.

    6-Tipos de bases de Datos, según:

    Variabilidad:

  • Bases de datos estáticas: Mantienen datos que no cambian frecuentemente.
  • Bases de datos dinámicas: Actualizan y modifican los datos regularmente.

    • Contenido:

  • Bases de datos textuales: Almacenan principalmente texto y documentos.
  • Bases de datos multimedia: Incluyen imágenes, audio y video.

    • Modelo de base de datos:

  • Bases de datos jerárquicas: Organizan los datos en una estructura de árbol.
  • Bases de datos de red: Permiten relaciones muchos a muchos entre entidades.
  • Bases de datos relacionales: Utilizan tablas relacionadas mediante claves.
  • Bases de datos orientadas a objetos: Almacenan datos complejos como objetos.

    • 7-Costo de licencia

    El modelo relacional es un modelo de datos para la gestión de bases de datos que organiza los datos en tablas (o relaciones) que consisten en filas y columnas. Cada tabla representa una entidad y cada columna representa un atributo de esa entidad. Las relaciones entre las tablas se establecen mediante claves primarias y claves foráneas.

    8-Costo de licencia

    Un Diagrama entidad-relación (DER) incluye los siguientes elementos principales:

  • Entidad: Representa un objeto o concepto en el mundo real, como una persona, lugar o cosa.
  • Atributo: Características o propiedades de una entidad que describen sus detalles.
  • Relación: Asociación o conexión entre entidades que describe cómo están relacionadas entre sí.
  • Clave primaria: Atributo o conjunto de atributos que identifican de manera única a cada instancia de una entidad.
  • Clave foránea: Atributo que establece una relación entre dos tablas al referenciar la clave primaria de otra tabla.

    • 9-¿Qué es una entidad?

    Una entidad es un objeto o concepto del mundo real que puede ser identificado mediante un conjunto de atributos. En el contexto de las bases de datos, una entidad se representa como una tabla en el modelo relacional.

    10-¿Qué es un atributo y sus tipos?

    Un atributo es una característica o propiedad que describe una entidad. Puede ser de diferentes tipos:

  • Atributo simple: Contiene un único valor para cada instancia de la entidad.
  • Atributo compuesto: Está formado por la combinación de varios atributos simples.
  • Atributo derivado: Su valor se calcula o deriva de otros atributos.
  • Atributo multivaluado: Puede tener varios valores para una misma instancia de la entidad.
  • Atributo clave: Identifica de manera única a cada instancia de la entidad.

    • 11-¿Qué es una relación?

    Una relación describe la asociación o conexión entre dos o más entidades en una base de datos. Las relaciones se establecen mediante claves primarias y foráneas que permiten vincular las filas de diferentes tablas. Pueden ser de diferentes tipos como uno a uno, uno a muchos o muchos a muchos, dependiendo de cómo se relacionan las entidades entre sí.

    12-Costo de licencia

  • Llave primaria (Primary Key): Es un atributo o conjunto de atributos que identifica de manera única a cada instancia de una entidad dentro de una tabla. Sus características son:
  • Debe contener valores únicos para cada fila.
  • No puede contener valores nulos (NULL).
  • Puede estar compuesta por uno o más atributos.


  • Llave foránea (Foreign Key): Es un atributo o conjunto de atributos en una tabla que hace referencia a la llave primaria de otra tabla. Sus características son:
  • Establece una relación entre las tablas.
  • Garantiza la integridad referencial, asegurando que no se puedan insertar valores que no existan en la tabla referenciada.
  • Puede aceptar valores nulos si se permite la relación opcional.

    • 13-Costo de licencia

    Las reglas del negocio son directrices o restricciones que definen cómo debe operar una organización, cómo deben realizarse ciertas tareas o cómo deben comportarse ciertos aspectos del negocio. Estas reglas son importantes para garantizar consistencia, precisión y conformidad dentro de una organización o sistema de información.

    14-Costo de licencia

  • Cardinalidad: En el contexto de los diagramas entidad-relación, la cardinalidad se refiere a la cantidad de ocurrencias de una entidad que pueden estar asociadas con otra entidad mediante una relación.
  • Uno a Uno (1:1): Cada instancia de una entidad se asocia con exactamente una instancia de otra entidad.
  • Uno a Muchos (1 ): Cada instancia de una entidad puede estar asociada con varias instancias de otra entidad, pero cada instancia de la segunda entidad solo puede estar asociada con una instancia de la primera entidad.
  • Muchos a Muchos (M ): Varias instancias de una entidad pueden estar asociadas con varias instancias de otra entidad.
  • Mapeo: Es el proceso de asignar o relacionar entidades y atributos de un modelo de datos a tablas y columnas en una base de datos relacional. Implica convertir el diseño conceptual (entidades, relaciones) en un diseño físico (tablas, columnas, claves).
