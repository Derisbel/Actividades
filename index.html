Elementos de Páginas Web

I. Orígenes de la Web

1. Web 2.0

La Web 2.0 se refiere a la evolución hacia un modelo más interactivo y colaborativo, promoviendo una mayor participación y democratización de la información.

2. Web 3.0

También conocida como la web semántica, busca mejorar la conectividad de datos y la inteligencia de las aplicaciones mediante tecnologías como IA y blockchain.

3. W3C y los estándares

La W3C desarrolla estándares abiertos para asegurar el crecimiento a largo plazo de la web, mejorando su interoperabilidad y accesibilidad.

II. Lenguajes de Etiquetas

1. SGML

Base de muchos lenguajes de marcado, permite definir la estructura de documentos.

2. XML

Permite definir estructuras de datos personalizadas y es ampliamente utilizado para el intercambio de datos.

3. HTML

Lenguaje de marcado estándar para crear páginas web, definiendo la estructura del contenido.

4. XHTML

Versión más estricta de HTML que sigue reglas de XML, mejorando compatibilidad.

5. HTML5

Última versión de HTML, ofrece nuevas características como soporte para multimedia y mejora en la semántica.

III. Otros Lenguajes Relacionados al Desarrollo Web

  • CSS/CSS3: Define la presentación visual de documentos HTML.

  • JavaScript: Lenguaje de programación del lado del cliente que permite interactividad.

  • Ajax: Técnica para actualizar páginas web asíncronamente.

  • Tecnologías del lado del servidor: Incluyen lenguajes como PHP, Python y Node.js.

  • Navegadores: Programas que permiten acceder y visualizar contenido web.

  • Motores renderizados: Interpretan y muestran el contenido de las páginas web.

IV. Herramientas de Desarrollo

1. Software WYSIWYG (What You See Is What You Get).

Estas herramientas permiten a los desarrolladores crear contenido web visualmente, sin necesidad de escribir código manualmente. Ejemplos incluyen: Adobe Dreamweaver y WordPress.

2. Otras alternativas.

  • Figma: Herramienta de diseño colaborativo que permite crear prototipos y interfaces de usuario.

  • InVision: Plataforma para diseñar, prototipar y colaborar en proyectos de diseño web.

  • Zeplin: Herramienta que facilita la colaboración entre diseñadores y desarrolladores al proporcionar especificaciones de diseño.

  • GitHub: Plataforma de desarrollo colaborativo que permite gestionar proyectos de software y controlar versiones de código.

3. Inspección de código desde el navegador.

La mayoría de los navegadores modernos incluyen herramientas de desarrollo que permiten inspeccionar, editar y depurar el código HTML, CSS y JavaScript de las páginas web en tiempo real. Esto es esencial para el desarrollo y la optimización de sitios web.

Tema 2: El Diseño De Una Página Web

I. Tipos de Páginas Web

1. Páginas estáticas.

Contenido fijo que no cambia a menos que se edite manualmente. Suelen ser simples y se utilizan para presentar información básica, como perfiles de empresa o portafolios.

2. Páginas dinámicas.

Contenido que cambia en función de la interacción del usuario o datos en tiempo real. Utilizan lenguajes del lado del servidor y bases de datos para ofrecer experiencias personalizadas.

3. Blogs: Actualización frecuente de contenido.

Páginas enfocadas en contenido actualizado frecuentemente, como artículos y entradas de opinión. Permiten interacción mediante comentarios y son una forma popular de compartir información.

4. Tiendas en línea: Diseñadas para la venta de productos.

Páginas diseñadas para la venta de productos o servicios. Incluyen catálogos, carritos de compra y sistemas de pago.

5. Redes sociales: Interacción y compartición de contenido.

Plataformas que permiten a los usuarios interactuar, compartir contenido y conectarse. Ejemplos incluyen Facebook, Instagram y Twitter.

6. Páginas de aterrizaje: Conversión de visitantes en clientes.

Diseñadas para convertir visitantes en clientes potenciales. Suelen tener un enfoque específico, como promociones o captación de correos electrónicos.

II. Puntos Básicos en la Elaboración de una Página Web

  • Objetivo claro: Definir el propósito de la página.

  • Investigación de audiencia: Comprender al público objetivo.

  • Estructura y navegación: Planificar una jerarquía clara.

  • Diseño responsivo: Adaptabilidad a diferentes dispositivos.

  • Contenido de calidad: Crear contenido relevante.

  • Optimización para SEO: Mejorar visibilidad en motores de búsqueda.

Tema 3: Estructura De Una Página Web

I. Jerarquía de Directorio de un Proyecto Web

Raíz del proyecto

Carpeta principal que contiene todos los archivos del sitio.

Subcarpetas comunes:

  • css: Archivos CSS
  • js: Archivos JavaScript
  • images: Imágenes utilizadas en el sitio.
  • fonts: Fuentes personalizadas
  • pages: Páginas HTML individuales
  • assets: Otros recursos como videos y documentos.

II. Elementos de Estructura para el Diseño de una Página Web

1. Regla utilizada para validar y certificar una página web

Se utilizan validadores como el W3C Markup Validation Service para comprobar que el código HTML y CSS cumple con los estándares establecidos. Esto asegura que la página funcione correctamente en diferentes navegadores y dispositivos.

2. Reglas para escribir HTML

  • Utilizar etiquetas correctamente anidadas.
  • Cerrar todas las etiquetas.
  • Usar atributos válidos y significativos.
  • Seguir una estructura semántica que mejore la accesibilidad y SEO.

3. Estructura básica: Incluye secciones head y body.

  • Esta estructura incluye las secciones fundamentales: head (metadatos y enlaces a recursos) y body (contenido visible).
